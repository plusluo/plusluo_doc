# OpenClaw 衍生 / 重写 / 替代项目一览

> 2026 年 2 月，OpenClaw 以 215K+ GitHub Stars 统治开源 AI Agent 赛道的同时，一批"重写 Claw"项目密集涌现。以下按切入路线分类汇总。

## 一、轻量化重写（"OpenClaw 太重了"）

| 项目 | 语言 | 代码量 | 核心主张 | 备注 |
|------|------|--------|---------|------|
| **NanoClaw** | Python | ~500 行 | 安全优先极简实现，砍掉非沙箱化风险 | 开发者 Gavriel Cohen，一周 7K+ Stars |
| **NanoBot** | Python | ~4K 行 | 保留 99% 核心功能（记忆/搜索/定时/多平台），部署 < 2 分钟 | 香港大学 HKUDS 实验室出品，卡帕西公开关注 |
| **ZeroClaw** | Rust | — | 100% Rust 重写，内存 < 5MB，毫秒级启动，单二进制分发 | 支持从 OpenClaw 一键迁移配置 |
| **PicoClaw** | Go | — | 跑在 $10 嵌入式硬件上，极致资源占用 | Sipeed 推出，面向 IoT/边缘场景 |
| **CountBot** | Python | ~21K 行 | 不追求更小，追求更适合中文用户 | 中文语义优化、本土平台优先 |

## 二、本土化封装（"OpenClaw 在国内水土不服"）

| 项目 | 厂商/团队 | 核心主张 | 备注 |
|------|----------|---------|------|
| **Molili（莫哩哩）** | 当贝科技 | OpenClaw 中文定制版，一键安装、全中文 UI、搭载 DeepSeek/MiniMax 国产模型，首个同时支持微信+钉钉+飞书+Siri 四端控制 | 8000+ Skill 技能库，号称"3 分钟上手" |
| **MaxClaw** | MiniMax | 基于 OpenClaw 的云端封装，免配置、免 API 费、一键调用 | 直接集成在 MiniMax Agent 网页端，定位"拎包入住的精装房" |
| **Kimi Claw Beta** | 月之暗面 | 原生集成 OpenClaw 生态，浏览器内运行，一键调用 ClawHub 5000+ 社区技能 | 40GB 云存储，用户也可接入自有 OpenClaw 实例 |
| **LobsterAI** | 网易 | 国产版 OpenClaw | 网易直接推出的本土化方案 |

## 四、最新动态：OpenFang

| 项目 | 信息 |
|------|------|
| **OpenFang** | 2026 年 2 月底宣布开源。定位为 OpenClaw 的又一重写/升级替代方案，具体技术路线和差异化细节待进一步披露 |

> **趋势观察**：OpenClaw 的"重写潮"呈现三条清晰路线——**极致精简**（NanoClaw/NanoBot）、**极致性能**（ZeroClaw/PicoClaw）、**本土适配**（Molili/MaxClaw/Kimi Claw）。本质上反映了 OpenClaw 43 万行代码的"大而全"架构在不同场景下的取舍：轻量派砍功能换部署速度，性能派换语言求资源效率，本土派加封装降使用门槛。无论哪条路线，都在验证 OpenClaw 定义的 Agent 范式（记忆 + 工具 + 多平台 + 主动调度）已成为事实标准。
